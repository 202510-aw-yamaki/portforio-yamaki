build 各フェーズの詳細

企画フェーズ 必須
どのようにアイデアを出し、企画をまとめたか

企画提案書をもとに、来店前のユーザーに対して「診断」ではなく「香りづくりの準備体験」を提供する方向性を定めました。オンラインで好みを可視化し、オフラインでスタッフと一緒に仕上げる二段階体験を軸に据え、香りに興味はあるものの専門知識がなく選び方に迷いやすい20代後半〜40代前半を主なターゲットとして整理しました。また、回答負荷を上げすぎずに予約まで進める導線と、店舗側が事前に好みを把握しやすい構成を企画段階で明確にしました。

設計フェーズ 必須
設計時に意識したこと、工夫した点

サイトマップとワイヤーフレームでは、トップページからアンケートSTEP1、STEP2、香りグラフ、予約、予約完了へと迷わず進める一方向の導線を設計しました。画面ごとに役割を分け、トップページでは期待形成と安心材料の提示、アンケートでは「ナビ / 回答 / 補足」の役割分担、グラフでは可視化と微調整、予約では傾向確認と入力を一画面で理解できる構成にしています。デザイン面では、静かな上質感、やわらかな期待感、清潔感をキーワードに、余白を広く取りつつ押しつけのないUIになるよう整理しました。

実装フェーズ 必須
実装で苦労した点、解決方法

実装では、HTML / CSS / JavaScript による画面構築に加え、Spring Boot / Thymeleaf / MyBatis / MySQL を用いたバックエンド連携まで対応しました。特に、STEP1・STEP2 の回答保持、graph 画面での結果保存、予約ページでの予約枠取得と予約登録、完了画面での再取得までを一連の流れとしてつなぐ点に苦労しました。そこで、回答途中は sessionStorage で一時保持しつつ、graph 以降は API と DB に保存する構成に整理し、予約 API・アンケート結果 API・予約完了再取得の仕様をそろえることで、フロントとバックエンドの接続を安定させました。さらに、MySQL 導入、Docker 対応、レスポンシブ調整まで進め、実運用を見据えた形に整えました。

smart_toy AIツールの活用

AI活用の概要 任意
ChatGPTなどのAIをどのように活用したか

活用した場面：

・企画段階：アイデア出し、ペルソナ設定のアドバイス

アイデアは自分の中に2案ありました。その中でどちらがより香水ワークショップに見合うのか？という部分でAIとの対話、講師のアドバイスを受け、診断風香りづくりの準備体験という形に落ち着きました。

ペルソナに関しては、企画案をもとに設定していただき、それをもとにさらに細部を詰めることができたと思います。

・設計段階：ページ構成や質問の流れを考える中で、どこまでを1ページに載せるべきか、どの順番ならユーザーが迷わず進めるかで悩みました。その際にAIとの対話を通じて、情報を詰め込みすぎず、トップページは期待形成、アンケートは回答への集中、予約ページは確認と入力に役割を分ける形で整理できました。自分の中で曖昧だった導線の優先順位を言語化できた点が大きかったです。

・実装段階：画面実装やAPI連携の途中で発生したエラーの原因整理、修正方針の比較、コード確認に活用しました。自分で内容を確認しながら取り入れることで、実装の抜け漏れを減らせたと思います。

・ドキュメント作成：仕様書や制作資料の文章整理、見出し構成、説明文のたたき台作成に活用しました。最終的な内容は自分で確認し、作品全体の流れに合うよう調整しています。
