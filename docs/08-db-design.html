3 インデックス設計

インデックス一覧 必須
テーブル名 インデックス / 制約名 対象カラム 種類 設計理由
visit_types UNIQUE(code) code UNIQUE 来店種別コードの重複防止
reservation_slots uq_reservation_slots_date_time slot_date, slot_time UNIQUE 同一日時の重複枠作成防止
reservations UNIQUE(reservation_code) reservation_code UNIQUE 完了画面再取得に使うコードの一意性確保
questionnaire_results UNIQUE(result_code) result_code UNIQUE アンケート結果再取得コードの一意性確保

補足: 「1日時点につき1件で受付終了」は現在アプリケーション側の重複予約防止と `reservation_slots.status=reserved` 更新で担保している。`reservations.reservation_slot_id` には現時点で DB 側の UNIQUE 制約は置いていない。

4 初期データ

マスタデータ・初期データ 必須
visit_types:
- workshop / 初回ワークショップ / 初回来店の香りづくり体験
- followup / 再来店相談 / 再来店時の調整・相談向け
- gift / ギフト相談 / ギフト用途の香り相談

reservation_slots:
- CURDATE() の翌日から 14 日分
- 1日あたり 3 枠
  - 10:30 recommended Haraguchi
  - 13:00 open Shimizu
  - 15:30 open Otsuka
- 既存同日時枠がある場合は INSERT しない
