1 プロジェクト振り返り

要件・設計面の振り返り 必須
うまくいったこと、難しかったこと、次回改善点を記載

要件面では、「診断」ではなく「来店前の香りづくり準備」として体験全体を設計できた点が良かったです。トップページで期待値を上げ、アンケートで負荷を抑え、グラフで納得感を作り、予約ページで来店意思を固める流れは、各画面の役割分担が明確でした。一方で、質問分岐、グラフ調整、予約導線をすべて連動させる設計は複雑で、仕様整理を先に固めないと実装判断がぶれやすい課題がありました。次回は、画面遷移図とデータの受け渡し仕様を初期段階でより明文化し、バックエンド保存前提の設計まで先に考えて作成に移る方がよかったと感じました。。

デザイン面の振り返り 必須
UI、ワイヤー、見せ方の観点で整理

デザイン面では、静かな高級感と親しみやすさを両立する方向性を各ページで維持できた点が成果でした。アンケートの選択肢画像や香りグラフの見せ方により、抽象的な香りの好みを直感的に扱いやすくできています。改善点としては、予約ページの設計が甘く、初見ユーザーには優先順位が分かりづらくなる可能性があります。重要CTA、入力欄、補足説明の視線誘導をさらに絞り、スマートフォンでの読みやすさも含めて情報整理を強めたいです。

フロントエンド実装面の振り返り 必須
HTML/CSS/JavaScript 実装の学びと反省

フロントエンド実装では、`sessionStorage` を使って複数画面のデータをつなぎ、状態を引き継ぐ構成をシンプルにまとめられた点が良かったです。特にアンケートから予約までをサーバー無しで成立させたことで、導線確認を素早く回せました。一方で、ロジックが各HTML内で分散しており、今後の拡張や保守では見通しが悪くなる懸念があります。全体の共通化分岐条件、最終的な設計までを先に用意しておくべきでした。

バックエンド実装面の振り返り 必須
Java / Spring Boot 前提での反省と改善

バックエンド実装面では、Spring Boot 導入前に必要な API 単位、DB 項目、状態遷移を先に文書化できた点は前進です。一方で、実装着手前の段階では Controller / Service / Mapper の責務分割、バリデーション方式、トランザクション設計をさらに具体化する必要があります。また、実装している途中で不足項目に気づくなど、本来は最初に全てを整えておくべきでした。

全体を通しての学び 必須
今回の制作で得た知見を簡潔に整理

今回の制作を通して、体験設計と画面実装は切り離せず、特に「どこで期待を上げ、どこで迷わせず、どこで納得させるか」を先に決めることが重要だと学びました。また、フロントだけで素早く体験を組み立てる有効性と、その一方でバックエンド連携を後回しにすると設計負債が生まれやすいことも実感しました。

2 技術記事

技術記事タイトル 必須

香りワークショップ制作におけるフロント主導の体験設計評価

記事本文 必須
第三者評価を想定した総評

来店前体験を複数画面で一貫したコンセプトのもとに構成し、ユーザーの完走率と予約導線を意識した設計ができている点は高く評価できます。特に、アンケートを単なる質問一覧ではなく、香り体験の準備として見せている点に企画的な強みがあります。一方で、データ永続化や自動テスト、運用時の予約管理など、実務運用に移るための基盤整備は今後の課題です。総じて、体験設計力とフロント表現力に強みがあり、次段階では実装基盤の強化が求められます。

4 今後の学習計画

今後学びたい課題・スキル 必須
次の制作に向けた具体的な学習計画

今後は、予約保存 API の実装、DB 設計の実装反映、入力バリデーション、自動テスト整備を優先して学びます。フロント面では、状態管理の整理、共通コンポーネント化、モバイルでの情報設計改善に取り組みます。あわせて、予約率や離脱率といった運用指標をもとに改善判断できるよう、簡易的な計測設計や分析視点も強化したいです。
