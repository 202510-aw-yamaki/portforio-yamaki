4 UIコンポーネント

ボタン 必須

主要ボタンはブラウン系背景に白文字、補助ボタンは淡色背景に濃色文字を使う。角丸は中程度、ホバー時は影か色差で反応を見せる。

フォーム要素 必須

選択肢カード、セレクト、スライダー、テキストエリアは余白を広めに確保し、入力中・選択中の状態がひと目で分かる配色を使う。

カード・テーブル・その他 必須

カードは淡色背景と控えめなシャドウで浮かせ、テーブルは情報を整理する用途に限定する。説明ブロックは余白を広く使い、圧迫感を避ける。

5 レイアウトルール

レイアウト基本ルール 必須

トップページは縦スクロールで期待形成を積み上げる。アンケートは「ナビ / 回答 / 補足」の役割分担を明確にし、予約ページは「傾向確認 / 入力 / 完了導線」を一画面で理解できる配置にする。スマートフォンではCTAと入力動線の見失いを避けるため、重要要素を縦に集約する。
