Webアプリケーション構造図 必須

管理システム（Spring Boot）

├─ ログイン前
│ ├─ /login（ログイン画面）
│ │ ├─ ID/メール入力
│ │ ├─ パスワード入力
│ │ └─ ログイン実行
│ └─ /logout-complete（ログアウト後画面）※任意
│
└─ ログイン後
├─ /admin（管理トップ）
│
├─ 地点管理（各地方）
│ ├─ /admin/locations（一覧・検索）
│ ├─ /admin/locations/new（追加）
│ ├─ /admin/locations/{id}/edit（編集）
│ └─ /admin/locations/{id}/delete（削除）
│
├─ 画像管理
│ ├─ /admin/images（一覧）
│ ├─ /admin/images/upload（追加）
│ └─ /admin/images/{id}/delete（削除）
│
├─ 公開設定
│ ├─ /admin/publish（公開/非公開）
│ └─ /admin/publish/order（表示順）
│
├─ 監査ログ
│ └─ /admin/audit-logs（操作履歴一覧・検索）
│
└─ 共通機能（全管理画面）
├─ ヘッダー（ユーザー情報、ログアウト、主要メニュー）
├─ ページネーション（一覧画面）
├─ バリデーション（入力チェック・エラー表示）
└─ 権限（Admin/Editor などのアクセス制御）

[ログイン前]
開始
↓
ログイン画面(/login)
├─ 認証成功 → 管理トップ(/admin)
└─ 認証失敗 → ログイン画面(/login)［エラー表示］


[ログイン後]
管理トップ(/admin)
├→ 地点管理 一覧・検索(/admin/locations)
│ ├→ 追加(/admin/locations/new) → 保存 → 一覧(/admin/locations)
│ ├→ 編集(/admin/locations/{id}/edit) → 保存 → 一覧(/admin/locations)
│ └→ 削除(/admin/locations/{id}/delete) → 実行 → 一覧(/admin/locations)
│
├→ 画像管理 一覧(/admin/images)
│ ├→ 追加(/admin/images/upload) → 保存 → 一覧(/admin/images)
│ └→ 削除(/admin/images/{id}/delete) → 実行 → 一覧(/admin/images)
│
├→ 公開設定(/admin/publish)
│ ├→ 公開/非公開 切替 → 公開設定(/admin/publish)
│ └→ 表示順設定(/admin/publish/order) → 保存 → 公開設定(/admin/publish)
│
├→ 監査ログ(/admin/audit-logs)
│ └→ 検索/絞り込み → 監査ログ(/admin/audit-logs)
│
└→ ログアウト(/logout) → ログイン画面(/login)


[共通機能の適用範囲]
ヘッダー（全管理画面）
ページネーション（/admin/locations, /admin/images, /admin/audit-logs）
バリデーション（追加/編集/公開設定の保存時）
権限制御（未認証: /loginへ、権限不足: 403）

3 ユーザーフロー

メインシナリオ 必須
ステップ 行動 表示ページ 次のアクション
1 「香り 地方 特産 香水」などで検索し、サイトを発見 Home（/index.html） ヒーローと地図を見てサイトの目的を把握する
2 予約CTAと日本地図を確認し、地方から探せることを理解 Home（/index.html） 興味のある地方（例：中部）をクリック
3 地方ページで香り素材の候補を一覧で確認 中部（/chubu/index.html） 気になる素材（例：能登ひば）を選択
4 素材の詳細情報を外部ページで確認し、会話ネタを収集 外部詳細ページ（WANOWA等） 他の素材も比較するため戻る
5 別地方の素材も見て、地域ごとの特徴を比較 近畿/四国など各地方ページ サロンの客層に合う香りテーマを絞る
6 店舗系の導線も確認し、体験先として妥当か判断 関東（/kanto/index.html） inimu店舗ページへ遷移
7 店舗情報・導線を確認し、実際に利用する意思決定 外部店舗ページ（https://inimu.jp/） 予約手続きへ進む
8 予約完了後、接客トークに使う「香り×地方」案を整理 （外部予約完了後） 来店前メモとして素材エピソードを準備
サブシナリオ 任意
その他の行動パターン（1〜2パターン）
ステップ 行動 表示ページ 次のアクション
1 SNSで「能登ひば」の投稿を見て、香りの背景を調べる 検索結果→Home（/index.html） サイト内で「中部」エリアを探す
2 地図から中部を選び、地域ごとの香り素材を確認 中部（/chubu/index.html） 能登ひばの詳細リンクを開く
3 外部詳細を読み、香りのストーリー性を把握 WANOWA詳細ページ（外部） 接客用に使える説明をメモする
4 比較のため別素材（国造ゆず・加子母ひのき）も確認 中部（/chubu/index.html） 季節提案に使えそうな素材を選ぶ
5 他地方の素材も見て会話の幅を広げる 四国/近畿ページ 「地域別おすすめトーク」を整理する
6 香り体験先として店舗情報を確認 関東（/kanto/index.html）→inimu（外部） 来店・予約の可否を確認する
7 予約を実施し、実体験を接客に活用する準備 inimu予約導線（外部） 次回来店客向けの提案メニューを作成する

4 URL設計

URL一覧 必須
URL HTTPメソッド 画面/機能 説明
/ GET Home トップページ（予約CTA＋全国地図）
/regions GET 地方一覧 地方の一覧（北海道〜九州）
/regions/{region} GET 地方ページ 各地方の拡大地図ページ（hokkaido など）
/spots GET 特産・地点一覧 全地方の地点/特産の一覧
/spots/{spot} GET 特産・地点詳細 各特産/地点の詳細（外部WANOWA等への遷移導線含む）
/brands GET ブランド一覧 ヘッダー導線用のブランド一覧
/brands/{brand} GET ブランドページ 各ブランドの遷移先ページ（内部/外部）
/legal/privacy GET プライバシー プライバシーポリシー
/legal/terms GET 利用規約 利用規約

Spring Bootアプリ部分（管理システム）

URL HTTPメソッド 画面/機能 説明
/login GET ログイン画面 認証フォーム表示
/session POST ログイン実行 認証してセッション作成
/session DELETE ログアウト セッション破棄
/logout-complete GET ログアウト完了 ログアウト後画面（任意）
/admin GET ダッシュボード 管理トップ

/admin/locations POST 地点追加実行 新規地点を作成
/admin/locations/{id} GET 地点詳細 地点詳細表示
/admin/locations/{id}/edit GET 地点編集画面 編集フォーム
/admin/locations/{id} PATCH 地点更新 地点情報を更新
/admin/locations/{id} DELETE 地点削除 地点を削除
/admin/images GET 画像一覧 画像リソース一覧
/admin/images/new GET 画像アップロード画面 画像登録フォーム
/admin/images POST 画像アップロード実行 画像作成
/admin/images/{id} GET 画像詳細 画像メタ情報表示
/admin/images/{id} DELETE 画像削除 画像削除
/admin/publish-settings GET 公開設定画面 公開/非公開・表示順設定
/admin/publish-settings PATCH 公開設定更新 公開状態や表示順を更新
/admin/audit-logs GET 監査ログ一覧・検索 操作履歴の参照（読み取り専用）
