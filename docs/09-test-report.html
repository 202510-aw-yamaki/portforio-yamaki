1 テスト方針

テスト対象・目的 必須

テスト対象はヘルスチェック API、予約枠取得 API、予約登録 API、予約完了画面再取得 API、アンケート結果保存 API、アンケート結果再取得 API です。目的は、DB を正とした予約導線が成立すること、完了画面の再取得前提が API レベルで成立すること、主要な異常系を統合テストで担保することです。自動テストは `SpringBootTest` + `TestRestTemplate` + `JdbcTemplate` による統合テスト構成です。
