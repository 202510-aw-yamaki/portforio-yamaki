パッケージの役割 必須

画面表示は Controller、JSON API は RestController、業務処理は Service、DB I/O は MyBatis Mapper、入出力定義は Form / DTO、永続化対象は Model に分離している。例外整形は advice、認証設定は config、業務例外は exception に集約する。

6 クラス間の依存関係

依存関係図 必須
FragrancePageController --> ReservationService --> VisitTypeMapper

QuestionnaireApiController --> QuestionnaireResultService --> QuestionnaireResultMapper

ReservationApiController --> ReservationSlotService --> ReservationSlotMapper
ReservationApiController --> ReservationService
                         --> ReservationMapper
                         --> ReservationSlotMapper
                         `-> VisitTypeMapper

StaffPageController --> StaffReservationService
                     --> ReservationMapper
                     --> VisitTypeMapper
                     `-> InstructorMapper

ApiExceptionHandler --> ErrorResponse
SecurityConfig --> Spring Security FilterChain / InMemoryUserDetailsManager
DI（依存性注入）の仕組み 任意

各 Controller / Service はコンストラクタインジェクションで依存を受け取る。公開 API 系では ObjectProvider を使い、Mapper 未構成時でもアプリを起動できるようにしている一方、スタッフ画面系は DB 前提の Mapper と ObjectMapper を直接注入している。
