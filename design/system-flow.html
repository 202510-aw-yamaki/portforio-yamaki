レイヤー構成の説明 必須

公開画面は FragrancePageController が Thymeleaf テンプレートを返し、公開データ取得・登録は QuestionnaireApiController と ReservationApiController が JSON API として提供する。スタッフ画面は StaffPageController が一覧、詳細、マスタ操作を担う。

DB アクセスは MyBatis Mapper に集約し、入力検証は @Valid と業務例外で補完する。例外応答は ApiExceptionHandler で統一し、スタッフ画面保護は SecurityConfig のフォームログインで行う。

2 リクエスト処理フロー

アンケート結果保存の処理フロー 必須
アンケート結果保存

1. フロントが POST /api/questionnaire-results を送信
2. QuestionnaireApiController が QuestionnaireResultForm を受領
3. Bean Validation を実行
4. QuestionnaireResultService.createResult(form)
5. resultCode を生成
6. questionnaire_results に JSON 列として保存
7. QuestionnaireResultResponse を 201 Created で返す
予約登録の処理フロー 必須
予約登録処理

1. フロントが POST /api/reservations を送信
2. ReservationApiController が ReservationForm を受領
3. Bean Validation を実行
4. ReservationService が slotId を日付と時刻へ分解
5. ReservationSlotMapper.findByDateAndTime() で枠を取得
6. VisitTypeMapper.findByName() で来店種別を取得
7. 枠状態が open / recommended か確認
8. ReservationSlotMapper.updateStatusIfAvailable(id, "reserved")
9. ReservationMapper.insert() で予約本体を保存
10. ReservationResponse を 201 Created で返す

5 認証・認可フロー

Spring Security の認証フロー 必須

公開ページと /api/** は匿名アクセスを許可し、/staff/** のみ ROLE_STAFF を要求する。スタッフ認証は InMemoryUserDetailsManager を使い、ユーザー名とパスワードは設定値から読み込む。

1. /staff/login に GET アクセス
2. ログインフォームを表示
3. /staff/login に POST
4. Spring Security が認証
5. 成功時は /staff/reservations
6. 失敗時は /staff/login?error
7. /staff/logout でログアウト
8. /staff/** 以外は現状 permitAll
